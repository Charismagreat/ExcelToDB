# 데이터 관리 원칙 (Data Management Principles)

이 프로젝트는 물리적 데이터베이스(Physical Table)와 시스템 메타데이터 가상 테이블(Virtual Table, `report_row`)의 이중 구조를 가집니다. 데이터의 정합성을 유지하기 위해 다음 원칙을 준수합니다.

## 1. 물리적 테이블 우선 반영 (Physical-First CRUD)

모든 데이터 변경(생성, 수정, 삭제) 작업은 반드시 **물리적 테이블에 먼저 반영**되어야 합니다.

- **이유**: 물리적 테이블은 시스템의 최종적인 진실의 원천(Source of Truth)입니다. 메타데이터 업데이트가 성공했더라도 물리적 테이블 반영에 실패하면 실제 데이터와 불일치가 발생합니다.
- **실행 순서**:
  1. 물리적 테이블(`tableName`)에 데이터 삽입/수정/삭제 수행.
  2. 물리적 작업 성공 시, 가상 테이블(`report_row`)에 해당 변경 사항 반영.
  3. 가상 테이블 업데이트 실패 시, 로그를 남기고 향후 동기화 작업을 통해 보정.

## 2. 정기적 동기화 (Regular Synchronization)

물리적 테이블과 가상 테이블 간의 데이터 건수나 내용이 불일치할 경우, 항상 **물리적 테이블을 기준**으로 가상 테이블을 보정합니다.

- 대시보드의 "Sync Status" 배지가 `Mismatch`를 나타낼 경우 동기화 스크립트를 통해 정합성을 맞춥니다.
- 가상 테이블에만 존재하고 물리적 테이블에는 없는 행은 '삭제됨(`isDeleted=1`)'으로 간주합니다.

## 3. 고유성 및 검증 (Uniqueness & Validation)

- 데이터 삽입 전 가상 테이블과 물리적 테이블 양쪽에서 중복 여부를 체크합니다.
- 필수 값(Required) 및 타입 검증은 물리적 테이블에 반영하기 직전에 서버 측에서 최종적으로 수행합니다.
