# 설계 문서: 사원용 영수증 AI 자동 입력 시스템 (Employee Receipt AI Input)

이 문서는 사용자가 사진을 찍으면 AI가 자동으로 테이블을 찾고 데이터를 채워주는 '마이 워크스페이스' 전용 Zero-UI 입력 시스템의 설계안입니다.

## 1. 개요
관리자가 엑셀로 생성한 '법인카드 영수증' 등 다양한 일반 테이블에 대해, 사원이 복잡한 타이핑 없이 **사진 촬영 + AI 분석 + 최종 확인**만으로 데이터를 정확히 입력할 수 있는 환경을 구축합니다.

## 2. 사용자 시나리오 (User Flow)
1.  **관리자**: 엑셀 업로드를 통해 '법인카드 영수증' 테이블 생성 (데이터ID, 날짜, 금액, 가맹점 등 포함).
2.  **사원**: '마이 워크스페이스' 앱 실행 -> 우측 하단 **Smart FAB** 클릭 -> 카메라 호출.
3.  **사진 촬영**: 영수증 사진 촬영 후 서버로 전송.
4.  **AI 로직 실행**: 
    - **테이블 찾기**: AI가 현재 사용 가능한 테이블 리스트 중 가장 적합한 테이블(예: 법인카드 영수증)을 자동 선택.
    - **데이터 추출**: 영수증에서 텍스트를 추출하여 JSON 구조화.
    - **시맨틱 매핑**: 추출된 데이터를 테이블 컬럼명(날짜/사용일자 등)에 맞춰 의미 기반으로 매핑 및 타입 변환(금액에서 '원' 제거 등).
5.  **사용자 확인 및 수정**: 오버레이 UI에 자동으로 채워진 입력 폼이 나타나며, 사원은 필요 시 수동으로 값을 수정함.
6.  **최종 저장**: '확인' 버튼 클릭 시 가상(`report_row`) 및 물리 테이블에 데이터가 동시 저장됨.

## 3. 상세 설계 (Technical Detail)

### 3.1 AI Vision & Table Discovery
- **Input**: 이미지(Base64) + 전체 테이블 목록(ID, 이름, 설명).
- **Process**: LLM(Vision 지원 모델)이 이미지의 성격(영수증, 명함, 보고서 등)을 파학하고 테이블 목록 중 가장 유사한 항목을 결정.
- **Output**: `best_match_table_id`, `confidence_score`.

### 3.2 Semantic Mapping & Type Conversion
- **추출 규칙**:
    - `금액`: 문자열에서 숫자만 추출하여 `number` 타입으로 정형화.
    - `날짜`: 다양한 형식의 날짜를 `YYYY-MM-DD` 표준 포맷으로 변환.
    - `가맹점/항목`: 시맨틱 유사도를 기반으로 가장 적절한 컬럼에 배정.
- **도구**: GPT-4o Vision 또는 전용 OCR + 정규화 프롬프트 라이브러리 활용.

### 3.3 UI/UX 구성 (AI Input Overlay)
- **Overlay Layer**: 기존 `AiInputOverlay.tsx`를 확장하여 '자동 채움 폼' 모드 추가.
- **Editable Fields**: 모든 데이터는 `Controlled Input`으로 구현하여 사원이 직접 수정 가능하도록 함.
- **Success Feedback**: 저장 완료 시 햅틱 피드백(터치 진동) 및 축하 애니메이션(Lottie/Framer Motion) 표시.

## 4. 데이터 안정성 및 보안
- **중복 방지**: `generateContentHash` 로직을 통해 동일 영수증의 중복 등록을 원천 차단.
- **권한 제어**: 사원(`VIEWER` 롤)은 본인이 작성한 데이터만 마이 워크스페이스 피드에서 확인 및 수정 가능.
- **로그 기록**: 모든 AI 입력 건에 대해 `changeType: 'AI_INPUT'`으로 히스토리에 기록.

## 5. 단계별 로드맵 (Roadmap)
- **Phase 1**: 사진 업로드 및 기본 OCR 데이터 추출 파이프라인 구축.
- **Phase 2**: 테이블 자동 매칭 및 컬럼 시맨틱 매핑 로직 완성.
- **Phase 3**: 사용자 수정 UI(오버레이) 및 최종 저장 프로세스 통합.
