# ExcelToDB: 스마트 자금일보 관리 시스템 디자인 문서

- **날짜:** 2026-03-28
- **상태:** 승인됨
- **기술 스택:** Next.js (App Router), SQLite, Prisma/Drizzle, Tailwind CSS

## 1. 개요
30명 규모의 중소기업에서 직원들이 각자의 엑셀 자금일보 양식을 업로드하고, 시스템이 이를 자동으로 분석하여 개별적인 CRUD 관리 화면과 통합 대시보드를 생성해 주는 웹 기반 솔루션입니다.

## 2. 핵심 요구사항
- **동적 엑셀 파싱:** 다양한 형태(컬럼 구성)의 엑셀 파일을 업로드 시 자동 처리.
- **개인화된 관리 권한:** 업로드한 사람이 해당 데이터의 소유자(관리자)가 되어 수정/삭제 권한 보유.
- **자동 UI 생성:** 업로드된 엑셀 구조에 맞춰 정렬/검색이 가능한 테이블과 입력 폼 자동 생성.
- **조회 전용 대시보드:** 소유자 외 직원들은 권한에 따라 리포트를 조회하고 시각화된 데이터를 확인.

## 3. 아키텍처 및 데이터 흐름

### 3.1 기술 스택
- **프레임워크:** Next.js 14+ (App Router)
- **데이터베이스:** SQLite (사내 서버 파일 기반 관리)
- **데이터 저장 방식:** JSON 컬럼을 활용한 스키마리스(Schemaless) 구조 대응
- **스타일링:** Tailwind CSS, Lucide Icons
- **차트:** Recharts 또는 Chart.js

### 3.2 데이터 모델 (핵심 테이블)
- **Users:** 사내 직원 정보 및 권한 (Admin/Viewer)
- **ReportMetadata:** 보고서 양식명, 시트명(sheetName), 소유자(ownerId), 컬럼 정의(JSON), 업로드 시간
- **ReportData:** 실제 데이터 내용 (JSON형태로 행별 저장)

### 3.3 주요 프로세스 및 멀티 시트/테이블 처리
1. **업로드 및 시트 추출:** 사용자가 엑셀 업로드 -> 서버에서 모든 시트 정보 추출 -> 사용자에게 등록할 시트 선택 화면 제공.
2. **멀티 테이블 감지:** 한 시트 내에 여러 표가 있을 경우, 빈 줄(Empty Row)을 기준으로 표 영역을 자동 분할하여 각각 독립적인 리포트로 등록 유도.
3. **UI 렌더링:** 메타데이터의 컬럼 정보를 읽어 동적으로 Table 및 Input Form 컴포넌트 생성.
4. **권한 체크:** 세션 정보를 확인하여 `ownerId`와 일치할 때만 수정/삭제 버튼 노출.

## 4. UI/UX 설계
- **Dashboard:** 전체 리포트 요약 및 공통 지표 시각화.
- **My Reports:** 내가 업로드한 보고서 목록 및 관리 버튼.
- **Report View:** 특정 보고서의 데이터 그리드 조회 및 상세 검색.
- **Report Edit:** 동적으로 생성된 폼을 통한 데이터 개별 수정/추가.

## 5. 보안 및 성능
- **인증:** 사내 접근 전용이므로 간단한 로그인 시스템 구축.
- **데이터 무결성:** 엑셀 업로드 시 데이터 타입 검증(금액, 날짜 등) 수행.
- **동시성:** 30명 규모이므로 SQLite의 동시 읽기/쓰기 성능으로 충분히 대응 가능.
