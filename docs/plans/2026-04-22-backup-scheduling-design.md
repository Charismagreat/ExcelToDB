# DB 백업 스케줄링 디자인 문서 (2026-04-22)

## 1. 개요
시스템의 안정적인 운영을 위해 데이터베이스 전체를 정기적으로 백업하고, 최신 10개의 스냅샷을 유지하는 자동화 시스템을 구축합니다.

## 2. 요구사항
- **주기**: 매주 월요일 ~ 토요일 (일요일 제외 6일)
- **시간**: 매일 새벽 3시 (03:00)
- **보관**: 수동/자동 구분 없이 총 10개의 백업 파일만 유지 (초과 시 가장 오래된 순으로 삭제)
- **제어**: 관리자 화면에서 스케줄링 활성/비활성 토글 가능

## 3. 기술적 설계

### 3.1 데이터 모델 (System Settings 추가)
`system_settings` 테이블에 다음 필드를 추가하여 설정을 관리합니다.
- `backupScheduleEnabled`: 스케줄링 활성화 여부 (INTEGER 0/1)
- `backupScheduleDays`: 실행 요일 (TEXT, "1,2,3,4,5,6")
- `backupScheduleTime`: 실행 시간 (TEXT, "03:00")
- `backupRetentionCount`: 보관 개수 (INTEGER, 기본값 10)

### 3.2 백업 스케줄러 (BackupScheduler)
- **라이브러리**: `node-cron` 사용
- **초기화**: Next.js의 `instrumentation.ts`를 사용하여 서버 시작 시점에 스케줄러 등록
- **동작**:
    1. 설정된 시간에 크론 잡 트리거
    2. `createBackupAction` 호출하여 백업 생성
    3. `backups/` 디렉토리 내 파일을 스캔하여 10개 초과 시 삭제

### 3.3 UI 구성 (BackupManager)
- **섹션**: "Auto-Backup Configuration" 섹션 추가
- **기능**:
    - 현재 스케줄 요약 표시 (예: "매주 월-토 03:00 실행 중")
    - 활성/비활성 토글 스위치 제공

## 4. 예외 처리 및 보안
- **서버 재시작**: 서버가 재시작되어도 `instrumentation.ts`를 통해 기존 설정대로 크론 잡이 다시 등록됨
- **권한**: 백업 관련 모든 액션은 `ADMIN` 권한 확인 필수
- **용량 관리**: 10개 제한을 엄격히 적용하여 디스크 부족 현상 방지

## 5. 검증 계획
- [ ] `node-cron`이 설정된 시간(테스트 시 1분 뒤 등)에 정상적으로 작동하는지 확인
- [ ] 10개 초과 시 가장 오래된 파일이 삭제되는지 확인
- [ ] 관리자 화면에서 토글 시 즉시 크론 잡이 등록/해제되는지 확인
