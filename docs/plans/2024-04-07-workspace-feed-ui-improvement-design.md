# 워크스페이스 피드 UI 개선 설계서 (2024-04-07)

본 설계서는 워크스페이스 메인 피드의 가독성과 정보 전달력을 높이기 위해, '테이블명 우선 표시' 및 '데이터 요약 중심 노출'을 골자로 하는 UI 개선안을 담고 있습니다.

## 1. 목적 및 필요성
- 기존 피드의 제목("신규 항목" 등)이 정적으로 고정되어 있어 수많은 항목 중 특정 데이터를 찾기 어려움.
- 본문이 상태 안내 위주로 구성되어 실제 데이터 내용(가게명, 금액 등)을 확인하려면 클릭이 필수적임.
- 사용자의 우선순위(2번 테이블명 -> 1번 데이터 내용 -> 3번 시간)를 반영하여 정보 계계를 재정립함.

## 2. 설계 상세

### 2.1 데이터 가공 로직 (`getWorkspaceFeedAction`)
피드 데이터를 생성하는 서버 액션에서 시각적 요소를 위한 필드를 보정합니다.

- **Title (제목)**: 
    - `reportId`가 있는 경우: `reportMap`에서 조회한 **테이블 이름**을 제목으로 사용.
    - `reportId`가 없는 경우: "미분류 항목" 또는 "분류 대기 중"으로 표시.
- **Content (내용)**: 
    - AI 분석 결과(`aiData`)가 있는 경우: 날짜, 주요 상호, 금액 등을 조합한 **데이터 요약문**을 생성하여 본문에 노출.
    - 예: `2024-04-07 스타벅스 15,000원`
- **Timestamp (시간)**: 기존 시간 표시 유지.

### 2.2 UI 컴포넌트 레이아웃 (`FeedCard.tsx`)
2번 시안(강조형 타이틀) 스타일을 적용합니다.

- **제목 영역**: 
    - 폰트: `text-base font-black text-gray-900`
    - 역할: 데이터의 분류(테이블 정체성)를 강력하게 명시.
- **본문 영역**:
    - 폰트: `text-sm font-semibold text-gray-700`
    - 역할: 사용자가 실질적으로 궁금해하는 데이터의 핵심 가치 전달.
- **안내 및 배지**:
    - 상태 기반 배지("완료", "분석중")는 썸네일 영역 상단에 유지하여 상태를 직관적으로 전달.

## 3. 예외 처리
- AI 분석이 실패하거나 데이터가 없는 경우: 기존의 안내 문구("분석 결과가 없습니다")를 유지.
- 테이블 이름이 너무 긴 경우: 제목 영역에 `line-clamp-1`을 적용하여 레이아웃 무너짐 방지.

## 4. 검증 계획
- [ ] 피드 목록 상단에 테이블 이름이 굵게 표시되는지 확인.
- [ ] 제목 바로 아래에 요약된 데이터가 정확히 노출되는지 확인.
- [ ] '미분류' 항목의 경우에도 정해진 규칙에 따라 제목이 표시되는지 확인.
