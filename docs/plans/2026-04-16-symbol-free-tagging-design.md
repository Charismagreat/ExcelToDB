# 디자인 문서: 기호 없는 지능형 태그 인식 (Symbol-free Intelligent Tagging)

- **상태**: 승인됨 (Approved)
- **작성일**: 2026-04-16
- **담당자**: Antigravity

## 1. 개요 (Overview)
현재 워크스페이스 입력창은 `@`(사람), `#`(조직/사물) 기호를 사용하여 마스터 데이터를 호출합니다. 본 설계는 이러한 명시적 기호 입력 없이도 일상적인 타이핑 과정에서 마스터 데이터를 실시간으로 감지하고 태그 전환을 제안함으로써 사용자 경험(UX)을 극대화하는 것을 목표로 합니다.

## 2. 요구사항 및 제약사항 (Requirements)
- **실시간 감지**: 사용자가 타이핑하는 동안 배경에서 실시간으로 마스터 데이터를 검색해야 함.
- **최소 호출 제한**: 한글 2글자, 영문 3글자 이상 입력 시에만 작동하여 불필요한 추천 팝업을 최소화함.
- **전체 카테고리 검색**: 기호가 없을 때는 사원, 거래처, 제품, 부서 등 모든 마스터 테이블을 통합 검색함.
- **세미 오토(Semi-auto) 전환**: 공백 입력 시 자동 변환하지 않으며, 사용자가 추천 리스트에서 선택하거나 Enter/Tab을 눌렀을 때만 칩으로 변환함.

## 3. UI/UX 설계 (Visual Design)
- **추천 팝업 차별화**: 기호 없이 발생한 추천창 상단에 "✨ 태그 인식" 헤더를 표시하여 시스템 제안임을 명시함.
- **일관성 유지**: 기존 칩(Chip) 스타일 및 팝업 레이아웃을 그대로 재활용하여 일관된 시각적 피드백 제공.
- **방해 금지**: 사용자가 추천을 무시하고 계속 타이핑하면 팝업이 자연스럽게 사라지도록 설계.

## 4. 기술 아키텍처 (Architecture)

### 4.1. 프론트엔드 (React)
- `AiInputOverlay.tsx`의 `handleEditorInput` 로직 확장:
    - 트리거 기호가 없는 경우, 현재 커서 앞의 마지막 단어를 추출.
    - 단어 길이가 임계값(한글 2자) 이상일 때 `searchAutocompleteTagsAction` 호출.
- `suggestionQuery` 및 `showSuggestions` 상태 공유.

### 4.2. 백엔드 (Server Actions)
- `searchAutocompleteTagsAction` 수정:
    - `trigger` 파라미터가 `null`인 경우 처리를 추가.
    - 모든 마스터 테이블(`master_client`, `master_product`, `user`, `department` 등)을 병렬 또는 순차 쿼리하여 상위 결과를 통합 반환.

## 5. 검증 계획 (Verification)
- **기능 테스트**: `@`나 `#` 없이 "삼성" 또는 "홍길" 입력 시 추천창이 정상적으로 뜨는지 확인.
- **성능 테스트**: 빠른 타이핑 시 서버 요청 부하가 적절한지 확인 (Debounce 적용 검토).
- **UX 테스트**: 의도치 않은 추천창 발생 빈도가 사용자 입력을 방해하지 않는 수준인지 검토.
