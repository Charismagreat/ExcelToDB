# [Design] 퍼블리싱 허브 전면 재구축 (마이크로 앱 스튜디오)

## 1. 목적 및 배경
기존 퍼블리싱 허브의 위저드 방식은 MY DB와의 데이터 정합성 유지에 어려움이 있었고, 단일 테이블 발행이라는 한계가 있었습니다. 이를 해결하기 위해 **"앱 프로젝트"** 기반의 멀티 테이블 발행 시스템으로 전면 재구축합니다.

## 2. 핵심 설계 원칙
1. **완벽한 데이터 정합성 (Perfect Parity)**: 데이터 미리보기를 복제하지 않고, 실제 MY DB 리포트 페이지(`/report/[id]`)를 새 탭으로 호출하여 100% 동일한 뷰를 보장합니다.
2. **멀티 테이블 지원**: 하나의 마이크로 앱에 여러 개의 데이터 소스(테이블)를 포함할 수 있는 구조를 채택합니다.
3. **직관적인 빌더 경험**: 사용자가 앱을 먼저 생성하고, 필요한 데이터를 "추가"해 나가는 스튜디오 방식의 UI를 제공합니다.

## 3. 주요 컴포넌트 및 구조

### 3.1. 퍼블리싱 허브 홈 (Hub Home)
- 제작 중인 앱/발행된 앱 목록 표시
- [새 마이크로 앱 만들기] 버튼 제공

### 3.2. 마이크로 앱 빌더 (App Builder)
- **Header**: 앱 기본 정보 (이름, 설명, 아이콘) 설정
- **Data Source Panel**: 앱에 포함된 테이블 목록 관리
  - 테이블 추가: 전체 테이블 목록(물리/금융/홈택스)에서 선택
  - 테이블 확인: 클릭 시 새 탭에서 `/report/[id]` 페이지 오픈
  - 테이블 삭제: 앱 프로젝트에서 해당 테이블 제외
- **Publishing Panel**: 최종 발행 버튼 및 권한 설정

### 3.3. 데이터 모델 (Schema)
- `micro_apps`: 앱 고유 정보 (id, name, description, ownerId, status)
- `micro_app_sources`: 앱과 테이블 간의 M:N 매핑 (appId, sourceTableId, displayName)

## 4. 사용자 흐름 (User Flow)
1. 사용자가 **[+ 새 마이크로 앱 만들기]** 클릭
2. 앱 이름(예: "자금 관리 대시보드") 입력 후 프로젝트 생성
3. **[데이터 소스 추가]**를 통해 '은행 거래 내역', '매출 세금계산서' 등 여러 테이블을 앱에 연결
4. 연결된 각 테이블의 **[데이터 확인]** 버튼을 눌러 새 탭에서 MY DB 정합성 최종 검토
5. **[최종 앱 발행]** 클릭 시 사용자 워크스페이스에 해당 앱이 정식 등록됨

## 5. 기대 효과
- MY DB와의 뷰 차이로 인한 사용자 혼선 원천 차단
- 복잡한 리포트 엔진 재구현 비용 제거 및 유지보수성 향상
- 실제 업무 환경에 필요한 멀티 테이블 기반 마이크로 앱 제공 가능
