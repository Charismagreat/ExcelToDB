# [Design] AI 이미지 일괄 분석 및 자동 입력 기능 (Magic Batch Fill)

## 개요
사용자가 업로드하거나 촬영한 여러 장의 이미지(영수증, 명함 등)를 최신 AI(Gemini 1.5 Flash)로 분석하여 현재 테이블 스키마에 맞는 데이터를 추출하고, 이를 일괄 검토 후 저장할 수 있는 지능형 입력 시스템을 구축합니다.

## 핵심 가치
1. **생산성:** 수동 입력 시간을 90% 이상 단축.
2. **정확성:** AI 추출 결과에 대한 시각적 강조와 사용자 검토 단계 제공.
3. **유연성:** 파일 업로드와 실시간 카메라 촬영 모두 지원.

## 상세 설계

### 1. 사용자 워크플로우 (UX Flow)
1. **진입:** 보고서 상세 페이지(`/report/[id]`) 내 '데이터 추가' 섹션 하단에 **[AI 사진으로 일괄 등록]** 버튼 배치.
2. **입력 옵션:** 파일 선택(Explorer) 또는 카메라 실행(Mobile) 선택.
3. **이미지 큐(Pre-processing):** 선택된 이미지들의 썸네일을 보여주며 '분석 대기' 상태 표시.
4. **AI 분석:** 
   - 각 이미지를 한 장씩 서버 액션으로 전송.
   - 현재 테이블의 `columns` 정보를 프롬프트에 포함하여 Gemini 1.5 Flash 호출.
   - 분석 결과(JSON) 수신 시 해당 행의 데이터를 즉시 채움.
5. **검토(Review Mode):**
   - 추출된 값은 **옅은 파란색 배경**과 ✨ 아이콘으로 강조.
   - 사용자가 값을 수정하면 강조 색상 제거.
   - 이미지 썸네일을 클릭하면 원본 사진을 크게 보여주어 대조 사격 지원.
6. **최종 저장:** [모두 저장하기] 버튼을 눌러 `addRowsAction` 호출 및 DB 반영.

### 2. 기술 아키텍처 (Architecture)

#### 2.1 주요 컴포넌트
- **`AIPhotoImportSection.tsx` [NEW]:** 이미지 선택, 분석 진행률, 리뷰 목록을 총괄하는 메인 컨테이너.
- **`ImageReviewRow.tsx` [NEW]:** 분석된 한 행의 데이터를 보여주고 수정을 지원하는 서브 컴포넌트.
- **`src/app/actions.ts` [MODIFY]:** `analyzeImageAndExtractDataAction` 추가.
- **`src/lib/ai-vision.ts` [MODIFY]:** `extractDataFromImage` (스키마 기반 추출 로직) 추가.

#### 2.2 데이터 구조
- **추출 프롬프트:** 
  ```text
  이미지에서 다음 항목들을 찾아 JSON 리스트 형태로 반환해줘.
  항목: [컬럼명1(타입), 컬럼명2(타입), ...]
  규칙: 
  - 날짜는 YYYY-MM-DD 형식.
  - 숫자는 쉼표 없이 숫자만.
  - 찾을 수 없는 정보는 null로 표시.
  ```

### 3. 예외 처리 (Error Handling)
- **분석 실패:** "텍스트가 흐릿하여 읽을 수 없습니다" 등 사용자 친화적 메시지 출력.
- **중복 입력:** 데이터 ID 규칙을 활용하여 중복 방지 (기존 `addRowsAction` 로직 활용).
- **타입 불일치:** 숫자 필드에 텍스트가 추출된 경우 빨간색 경고 표시 및 수동 수정 유도.

### 4. 검증 계획 (Verification)
1. **기능 테스트:** 3장 이상의 영수증 사진을 한 번에 올렸을 때 각 행이 올바르게 생성되는지 확인.
2. **UI 테스트:** AI가 채운 필드의 배경색 강조 및 수정 시 색상 제거 동작 확인.
3. **모바일 테스트:** 카메라 촬영 후 즉시 분석 및 입력이 가능한지 확인.
